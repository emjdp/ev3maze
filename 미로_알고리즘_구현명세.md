# 미로 라인트레이서 — 알고리즘 & 구현 명세서

3개 컬러센서로 미로의 모든 노드를 지나 도착(7)에 가고, 다시 모든 노드를 지나 출발(0)로 돌아오는 로봇.
EV3 Lab · Python · 블록코딩(앱인벤터/스크래치) **세 플랫폼 어디서든** 구현할 수 있도록, 자료구조·판단·기록 방식을 모두 풀어 쓴다.

---

## 0. 한눈에 보는 전체 그림

1. **EXPLORE(탐사 주행)**: 출발에서 좌선우선법으로 라인을 따라가며, 분기를 만나면 정해진 규칙으로 방향을 고른다. 고른 **회전 동작을 배열에 차곡차곡 기록**한다. 잎(막다른 길)에서는 색을 읽고 U턴한다. 모든 노드를 지나 도착(7)에서 끝난다.
2. **RETURN(복귀 주행)**: 기록 배열을 **거꾸로 + 좌우 반전**해서 재생한다. 라인은 센서로 계속 따라가되, 분기에서의 회전만 기록대로 한다(다시 탐색·스캔하지 않음). 모든 노드를 지나 출발(0)에서 끝난다.

핵심 아이디어 4가지:
- **좌선우선법(Left-hand rule)**: 분기에서 항상 "좌 > 직진 > 우 > U턴" 순으로 미탐색 방향을 고른다.
- **분기 형태 peek**: 좌·우가 동시에 보이는 분기(B·D)는 살짝 직진해 중앙센서로 확인 → 뚫리면 십자(D형), 막히면 T(B형). **분기마다 최초 1회만**.
- **한 분기 우선**: 한 분기의 탐색을 끝내기 전에는 다른 분기로 넘어가지 않는다(E에서 F를 보면 일단 F까지 가서 유턴, E의 5를 먼저 처리).
- **경로 기록/역재생**: 회전을 배열에 저장 → 복귀 때 거꾸로 재생하므로 길을 다시 헤매지 않는다.

## 1. 자료구조 — 무엇을 배열에 어떻게 저장하나

### 1-1. 회전 기록 배열 `path[]` (가장 중요)
분기/잎에서 **실제로 한 회전 동작**을 숫자 하나로 저장한다.

| 값 | 의미 |
|---|---|
| 1 | 좌회전 (L) |
| 2 | 직진 (S) |
| 3 | 우회전 (R) |
| 4 | U턴 (잎에서 되돌기) |

- `path`는 **1차원 정수 배열** 하나면 된다. (EV3 Lab은 2차원·객체를 못 쓰므로 이 단일 배열 방식이 정답)
- `pathLen`(현재 길이/쓰기 위치) 변수를 같이 둔다. 분기/잎에서 회전할 때마다 `path[pathLen] = 동작값; pathLen = pathLen + 1`.
- 예) 이 미로 EXPLORE 경로 `0,A,1,A,B,2,B,D,4,D,3,D,E,F,E,5,E,F,6,F,7`에서 각 결정 지점의 동작이 순서대로 쌓인다.

### 1-2. 분기 메모리 `seen[]` (왔던 분기 재탐색·재peek 방지)
"이 분기는 이미 와봤다"를 기록해 **peek를 분기당 1회**로 막는다.

- 분기는 5개(A,B,D,E,F). 각 분기에 **번호 0~4**를 부여하고 `seen[id]`에 0/1 저장.
- "현재 분기 번호"는 분기를 처음 만나는 순서대로 카운터(`jcount`)로 매기거나, 좌표가 있으면 좌표로 식별한다. 센서만 쓰는 로봇은 보통 **DFS 스택**(아래 1-3)으로 현재 분기를 안다.
- peek 직전: `if (seen[현재분기]==0) { peek 실행; seen[현재분기]=1 }` 아니면 peek 생략.

> EV3 Lab에서 분기당 여러 칸을 쓰고 싶으면 한 배열을 **오프셋**으로 나눈다. 예: `mem[]` 한 개를 분기당 3칸으로 보고 `idx = 분기번호*3 + off` (off 0=seen, 1=형태(B/D), 2=탐색마스크). 2차원 대신 이 인덱싱이 표준이다.

### 1-3. DFS 스택 `stack[]` (한 분기 우선 + 되돌아오기)
"한 분기를 끝내기 전엔 다른 분기로 안 간다"와 "왔던 길로 안 돌아간다"를 보장하는 장치.

- 분기에 **들어갈 때 push**, 그 분기의 모든 출구를 끝내면 **pop**(부모로 복귀).
- 스택 맨 위 = 지금 탐색 중인 분기. 잎에서 U턴해 돌아오면 스택 맨 위 분기로 돌아온 것이므로, 그 분기의 **다음 미탐색 출구**를 고른다.
- 이 스택 덕분에 로봇은 "이미 한 출구"를 다시 가지 않는다(아래 2-4 마스크와 함께).

## 2. 판단 로직 — 센서를 어떻게 읽고 어떻게 결정하나

### 2-1. 컬러센서 3개 저장·판독
- 센서는 **좌(L)·중(C)·우(R)** 3개. 각 센서를 **반사광(0~100)**으로 읽는다. 0=검은선, 100=흰바닥.
- 매 순간 3개를 읽어 **임계값(예: 40)** 으로 0/1 비트로 바꾼다: `bit = (반사광 < 40) ? 1 : 0`.
- 세 비트를 묶어 **패턴 숫자**로 만든다: `pattern = bitL*100 + bitC*10 + bitR` → 010, 110, 101, 111 …
- 저장: 변수 3개(`sL,sC,sR`)에 비트를 넣거나, `pattern` 하나로 묶어 쓴다. (플랫폼별로 1-1의 path와 똑같이 **숫자 변수**면 충분)

```
읽기 의사코드:
sL = (Color1.reflected < TH) ? 1 : 0
sC = (Color2.reflected < TH) ? 1 : 0
sR = (Color3.reflected < TH) ? 1 : 0
pattern = sL*100 + sC*10 + sR
```

### 2-2. 라인 추종(직진 구간) — 비례 제어
분기가 아닌 동안은 중앙센서로 라인을 따라간다.

```
error = TH - Color2.reflected      // 선 중앙에서 벗어난 정도
steering = Kp * error              // Kp는 0.5~1.5 정도 튜닝
MoveSteering(power=기본속도, steering)
```
좌·우 센서는 **분기 감지용**: 평소 둘 다 0(흰), 분기에서 한쪽/양쪽이 1(검)이 되면 "결정 지점 도착".

### 2-3. 회전 실행 — 라인 재포착 방식(자이로 불필요)
방향이 정해지면 **그 방향으로 제자리 회전하다가 중앙센서가 다시 선을 잡으면 정지**한다. 드리프트가 없고 가장 단순하다.

```
TurnUntilLine(dir):
  dir=좌: 좌로 제자리 회전
  dir=우: 우로 제자리 회전
  dir=직진: 짧게 직진
  dir=U턴: 제자리 180도(첫 선은 통과, 두 번째 선에서 정지)
  → 회전 중 Color2가 선(검)을 다시 잡으면 Brake
```

### 2-4. 분기별 대응 규칙 (좌선우선 + 형태 peek + 한 분기 우선)

분기에 도착하면 패턴을 읽고 아래 순서로 처리한다.

**(가) 형태 peek — 좌·우 둘 다 보이는 분기(패턴 1?1), 최초 1회만**
- `seen[분기]==0` 이면: **살짝 직진**해서 중앙센서 확인.
  - 직진이 **뚫림(010)** → **D형(십자)**: 이왕 온 김에 **직진부터** 탐색.
  - 직진이 **막힘(000)** → **B형(T)**: 후진해서 분기점 복귀 → 좌선우선.
  - `seen[분기]=1`로 표시. (다음에 이 분기로 되돌아오면 peek 생략)
- `seen[분기]==1` 이면(재방문): peek 안 함. 스택/마스크가 가리키는 **다음 미탐색 출구**로 바로.

**(나) 좌선우선 선택**
- 미탐색 출구 중 **좌(1) > 직진(2) > 우(3)** 순으로 하나 고른다.
- 잎으로 가는 출구든 분기로 가는 출구든, **좌선우선 순서대로** 고른다.

**(다) 한 분기 우선 — 다른 분기를 먼저 탐색하지 않기**
- 고른 방향으로 갔더니 **또 다른 분기**가 나오면(예: E에서 좌=F로 갔는데 F가 분기): 거기서 **유턴해 원래 분기로 복귀**하고, 원래 분기의 **남은 출구(잎)부터** 처리한다.
- 원래 분기의 잎 출구를 다 끝낸 뒤에야 그 "분기로 이어지는 출구"로 내려간다.
- 효과: 깊은 쪽(도착 7이 있는 스파인)이 자연히 **마지막**이 되어, **전 노드를 지난 뒤 7**에 도달한다.

**(라) 잎(막다른 길) — 패턴 000**
- 색을 읽어 체크포인트 기록(필요 시), **U턴(4)** 하고 부모 분기로 복귀.

### 2-5. 이 미로에서 실제로 일어나는 일 (검증용 트레이스)
경로: `0 A 1 A B 2 B D 4 D 3 D E F E 5 E F 6 F 7`

- **A**(좌+직, 110): 좌선우선 → 좌(1, 잎) → 복귀 → 직진(B). peek 없음.
- **B**(좌+우, 101): **최초 peek** → 직진 막힘(000) → B형 → 후진 → 좌(2, 잎) → 복귀 → 우(D).
- **D**(좌+직+우, 111): **최초 peek** → 직진 뚫림(010) → D형 → 직진(4, 잎) → 복귀 → (헤딩 뒤집힘) 좌(3, 잎) → 복귀 → 남은 출구(E).
- **D 재방문**(4·3에서 복귀): `seen[D]=1` → **peek 생략**.
- **E**(좌+직, 110): 좌선우선 → 좌(F)로 **끝까지 가봄** → **F가 분기** → **유턴해 E 복귀** → 직진(5, 잎) → 복귀 → 남은 출구(F).
- **F**(좌+직, 110): 좌(6, 잎) → 복귀 → 직진(7=도착). 끝.

→ 도착 7 직전 방문 8/8, 단일 패스(사전 학습 0).

## 3. 기록 & 역재생 — 어떻게 저장하고 어떻게 되돌아오나

### 3-1. 기록(EXPLORE)
분기/잎에서 회전할 때마다 그 동작값을 `path`에 append.
```
회전 실행 직후:
  path[pathLen] = 동작값(1/2/3/4)
  pathLen = pathLen + 1
```
직진(라인 추종)만 하는 구간은 기록하지 않는다(센서가 알아서 따라감). 분기·잎의 **회전만** 기록한다.

### 3-2. 역재생(RETURN)
도착(7)에서 시작. `path`를 **뒤에서 앞으로** 읽으며 **좌우를 반전**해 회전한다. 직진 구간은 센서로 계속 추종.
```
for k = pathLen-1 downto 0:
    동작 = path[k]
    반전: 1↔3 (좌↔우), 2는 그대로, 4(U턴)는 그대로
    LineFollow()  // 다음 결정 지점까지 라인 추종
    TurnUntilLine(반전된 동작)   // 스캔/peek 없이 확신 회전
출발(0) 마커를 만나면 정지.
```
- **왜 거꾸로+반전인가**: 갔던 경로를 그대로 되짚으려면 순서를 뒤집고, 진행 방향이 반대가 되므로 좌/우가 바뀐다. U턴과 직진은 방향 무관이라 그대로.
- **왜 다시 안 헤매나**: 분기에서 새로 판단·peek·스캔을 하지 않고 **기록값대로만** 회전하므로, 왔던 길을 정확히 역행하며 모든 노드를 다시 지나 0에서 끝난다.

### 3-3. 인덱스 내림차순 만들기 (플랫폼 공통 팁)
대부분의 루프는 0부터 증가한다. 역순 인덱스는 수식으로 만든다.
```
i = pathLen - 1 - 루프카운터
동작 = path[i]
```

## 4. 플랫폼별 구현 매핑

같은 알고리즘을 세 환경에서 어떻게 옮기는지 대응표.

| 개념 | EV3 Lab (EV3-G) | Python (예: pybricks/ev3dev) | 블록코딩(앱인벤터/스크래치) |
|---|---|---|---|
| 숫자 배열 `path` | 🔴Variable(Numeric **Array**) + Array Operations(Write/Read at Index, Length) | `path = []`, `path.append(x)`, `path[i]` | 리스트 블록(`리스트에 항목 추가`, `리스트의 i번째`) |
| 길이/포인터 | Numeric 변수 `pathLen` | `len(path)` | `리스트 길이` 블록 |
| 분기 메모리 `seen` | 같은 배열 오프셋 인덱싱(`j*5+off`) | `seen = {}` 또는 리스트 | 리스트/딕셔너리 블록 |
| 패턴 판단 | 🟠Switch(**Numeric 모드**, 입력=pattern) | `if pattern == 110:` … | `만약 pattern = 110` 조건 블록 |
| 센서 읽기 | 🟡Color Sensor(Reflected) | `ColorSensor.reflection()` | 센서 값 블록 |
| 라인 추종 | 🟢Move Steering + 🔴Math(비례) | `robot.drive(speed, Kp*error)` | 모터 출력 + 변수 연산 블록 |
| 회전 | 🟢Move Tank + 🟡Wait(Color<TH) | `while sensor.reflection()>TH: turn()` | `~까지 반복` + 모터 블록 |
| 반복 | 🟠Loop(Count/Until) | `for`/`while` | `반복`/`~까지 반복` |
| 함수화 | 🔵My Block(입력/출력 매개변수) | `def follow():` / `def turn(dir):` | 함수(블록) 정의 |
| 역순 인덱스 | 🔴Math `i=Len-1-index` | `for k in range(len(path)-1,-1,-1)` | `i = 길이 - 1 - 카운터` |

### 4-1. EV3 Lab 핵심 메모
- 2차원 배열/객체 없음 → **단일 숫자 배열 + 오프셋 인덱싱**.
- 데이터 와이어가 복잡해지면 다운되므로 상태는 **Variable 읽기/쓰기**로 주고받는다.
- 다중 분기는 **Switch를 Numeric 모드**로.
- Array Operations 블록의 **Write at Index**로 `path` 갱신, **Length**로 길이.
- 반복 동작은 **My Block**(Follow_Line, Read_Pattern, Peek_Check, Turn_To_Exit 등)으로 묶어 가독성↑. (My Block은 입력/출력 매개변수 지원, 한 프로그램에 32개 이내 권장)

### 4-2. Python 골격 (가장 읽기 쉬운 기준 구현)
```python
TH = 40
path = []

def read_pattern():
    sL = 1 if c1.reflection() < TH else 0
    sC = 1 if c2.reflection() < TH else 0
    sR = 1 if c3.reflection() < TH else 0
    return sL*100 + sC*10 + sR

def follow_until_event():
    while True:
        p = read_pattern()
        if p != 10:              # 010이 아니면 분기/잎
            return p
        err = TH - c2.reflection()
        robot.drive(SPEED, Kp*err)

def turn_until_line(d):          # d: 'L','S','R','U'
    start_pivot(d)
    while c2.reflection() > TH:   # 라인 재포착까지
        pass
    robot.stop()

def peek_is_cross():             # 살짝 직진 후 중앙 확인
    nudge_forward()
    cross = c2.reflection() < TH  # 직진 뚫림?
    if not cross: nudge_back()
    return cross

seen = {}
def explore():
    while not at_goal():
        p = follow_until_event()
        if p == 0:                      # 막다른 길(000)
            read_color(); record('U'); turn_until_line('U')
        else:
            j = current_junction()
            if has_left_and_right(p) and not seen.get(j):
                cross = peek_is_cross(); seen[j] = True
                # cross면 직진부터, 아니면 후진 후 좌선우선
            d = pick_left_first(p)       # 좌>직>우 미탐색
            record(turn_token(d)); turn_until_line(d)

def record(tok):                 # tok: 'L'/'S'/'R'/'U' → 1/2/3/4
    path.append({'L':1,'S':2,'R':3,'U':4}[tok])

def return_run():
    for k in range(len(path)-1, -1, -1):
        d = path[k]
        d = {1:'R',3:'L',2:'S',4:'U'}[d]   # 좌우 반전
        follow_until_event()
        turn_until_line(d)
    stop()
```

### 4-3. 블록코딩(앱인벤터/스크래치) 메모
- 배열 = **리스트 블록**. `path`에 `리스트에 항목 추가`, 읽기 `리스트의 i번째 항목`.
- 패턴 판단 = `만약/아니면` 중첩 또는 패턴 숫자 비교.
- 역재생 = `i를 (리스트 길이 - 1)부터 0까지 1씩 감소 반복`.
- 모터·센서는 각 환경의 확장(EV3/마이크로빗 등) 블록 사용. 알고리즘 흐름은 위 Python과 동일.

## 5. 의사코드 총정리 (플랫폼 무관)

```
초기화: path=[], pathLen=0, seen=모두0, stack=[]

EXPLORE:
  반복(도착 7 도달까지):
      pattern = 센서3개_읽기()
      만약 pattern == 010:         # 직선
          라인추종_한스텝()
      아니면 pattern == 000:       # 막다른 길(잎)
          색읽기(); 기록(U턴=4); 회전(U턴)
      아니면(분기):
          만약 (좌·우 둘다) 그리고 seen[분기]==0:
              형태 = peek_직진확인()      # 010=십자 / 000=T
              seen[분기] = 1
          dir = 좌선우선_미탐색_고르기()   # 좌>직>우
          만약 그 방향이 또 분기로 이어지면:
              그 분기까지 가서 유턴, 원래 분기 잎부터 먼저 처리
          기록(dir 토큰); 회전(dir)

RETURN:
  i = pathLen-1 부터 0까지 감소 반복:
      dir = 반전(path[i])           # 좌↔우, 직/유턴 유지
      라인추종_다음분기까지()
      회전(dir)                     # peek·스캔 없이 확신 회전
  출발 0에서 정지
```
